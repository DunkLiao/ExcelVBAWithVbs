Project layout
==============

src\vba\modules\ReportAutomation.bas
    Main VBA module stored as exported source.

scripts\bootstrap-report-workbook.vbs
    Creates a macro-enabled workbook and imports the VBA module.

scripts\run-sales-report.vbs
    Opens the workbook, loads a CSV file, and runs the VBA report macro.

scripts\export-vba-modules.vbs
    Exports standard/class/form VBA modules from the workbook back into the repo.

sample-data\sales-input.csv
    Example input file for the reference report flow.

Reference workflow
==================

1. Create the workbook:
   cscript //nologo scripts\bootstrap-report-workbook.vbs

2. Run the sample report:
   cscript //nologo scripts\run-sales-report.vbs sample-data\sales-input.csv

3. Export VBA changes back into the repo after editing the workbook in Excel:
   cscript //nologo scripts\export-vba-modules.vbs

Notes
=====

- Excel desktop must be installed.
- "Trust access to the VBA project object model" must be enabled in Excel for the bootstrap/export scripts.
- The export script currently skips workbook/sheet document modules unless you extend it for event-driven code.
- The reference CSV schema is:
  Team,Amount
- The VBA module creates or refreshes two sheets:
  InputData
  Report
